# RISC-V Single Cycle Processor on Basys 3 FPGA

## Overview

This project presents the design and implementation of a **32-bit Single Cycle RISC-V Processor** using **Verilog HDL**. The processor supports the core RV32I instruction set, including arithmetic, logical, immediate, branch, jump, load, and store operations.

The processor was designed following the standard single-cycle architecture where each instruction completes execution within a single clock cycle. The design consists of modular **datapath** and **control-path** components, enabling instruction fetch, decode, execution, memory access, and write-back operations.

Each module was individually verified through simulation and functional testing before system-level integration. The complete processor functionality was validated by successfully executing a **Fibonacci Series** program on a **Basys 3 FPGA** board and displaying the generated sequence on the onboard seven-segment display.

## Processor Architecture

The processor follows the standard RISC-V single-cycle datapath.

### Instruction Fetch (IF)

The Program Counter (PC) generates the address of the current instruction. 

The instruction memory is accessed using this address, and the fetched instruction is forwarded to the decode stage.

### Instruction Decode (ID)

The instruction is decoded to determine its type and required control signals. 

Source register operands are read from the register file while immediate values are generated by the immediate extension unit.

### Execute (EX)

The Arithmetic Logic Unit (ALU) performs arithmetic, logical, comparison, and address calculation operations according to the decoded instruction.

### Memory Access (MEM)

Load and store instructions access data memory using addresses generated by the ALU.

### Write Back (WB)

Results from either the ALU or data memory are written back into the register file.

## Design Modules

### Datapath Components

#### program_counter.v

Maintains the current instruction address and updates the Program Counter every clock cycle.

#### instruction_memory.v

Stores machine instructions and provides instruction data based on the Program Counter value.

#### registerfile.v

Implements thirty-two 32-bit general-purpose registers supporting simultaneous read and write operations.

#### immediate_extender.v

Extracts and sign-extends immediate values for various RISC-V instruction formats.

#### alu.v

Performs arithmetic, logical, comparison, and address-generation operations.

#### alu_mux.v

Selects the appropriate ALU operand based on instruction type and control signals.

#### pcmux.v

Selects the next Program Counter value between sequential execution and control-flow targets.

#### pcplus4.v

Computes the next sequential instruction address (PC + 4).

#### pctarget.v

Generates branch and jump target addresses.

#### resultmux.v

Selects the correct write-back source between ALU results and memory data.

#### data_memory.v

Implements memory access functionality for load and store instructions.

---

### Control Path Components

#### instruction_decoder.v

Decodes instruction fields required for datapath operation.

#### main_decoder.v

Generates high-level control signals based on instruction opcode.

#### alu_decoder.v

Determines the required ALU operation from instruction fields and control signals.

#### controlunit.v

Combines decoder outputs to generate all processor control signals.

---

### FPGA Interface Components

#### binary_to_bcd.v

Converts binary processor outputs into BCD format for display purposes.

#### seven_seg_display.v

Drives the Basys 3 onboard seven-segment display.

#### clock_divider.v

Generates slower clock frequencies required for display and hardware demonstration.

---

### Top-Level Integration

#### final_module.v

Integrates processor datapath, control path, memory, and display subsystems.

#### top_module.v

Top-level FPGA module used for synthesis and hardware deployment on the Basys 3 board.

To validate complete processor functionality, a Fibonacci Series program was executed on the implemented RISC-V processor.

The demonstration verified:

* Correct instruction fetch and decode
* ALU operation
* Register file functionality
* Branch and jump execution
* Memory access operations
* Program flow control
* Successful hardware deployment

The generated Fibonacci sequence was displayed on the onboard seven-segment display, confirming correct end-to-end processor operation on FPGA hardware.
